Java Container HashMap(use and basic theory)

The use of the HashMap

oldValue will return “2”, so it actually when we overwrite the same key, put() will return the original value.

The implementation of HashMap(JDK 7 and JDK 8)

It is displayed by an array and a linked list.

First if we want to store something into a array, we need a index. So where the HashMap get the index.

We can see every key will get a hash by using hash(key) and then use indexFor(), this i is the index we need.

A little bit more detail

it will return 115026, it’s too big for a index. So we need to do some operations to decrease this value.

So one method is Module Operation,

the index is the index we need.

But when we have a new key like key2, the index after module Operation is same as the key1.

When this situation happens, we will need to use linked list.

We use linked list to connect the key1’s value and key2’s value.

But when we have a key3 that has same index, what should we do now?

Actually, we need add it as the head of linked list. It will save a lot of time, since we don’t need traverse any item during the linked list.

And we need move key3 to the table, so that we can find key1, key3 and key2 now.

Now, we know the header of linked list is actually in the array, this is why we use header = table[i]
